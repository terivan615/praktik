[ART: A Unified Unsupervised Framework for Incident Management in Microservice Systems](https://dl.acm.org/doi/abs/10.1145/3691620.3695495)
---

### 1. Общая информация
*   **Название:** ART: A Unified Unsupervised Framework for Incident Management in Microservice Systems (ART: Единая неконтролируемая платформа для управления инцидентами в микросервисных системах).
*   **Авторы:** Исследователи из Нанкайского университета (Китай), Microsoft Research и Университета Цинхуа.
*   **Конференция:** ASE '24 (39-я Международная конференция по автоматизированной разработке программного обеспечения).
*   **Дата публикации:** Октябрь 2024 г.

### 2. Постановка проблемы
Управление инцидентами в крупных микросервисных системах — сложная задача, включающая три основных этапа:
1.  **Обнаружение аномалий (Anomaly Detection, AD):** Выявление сбоев.
2.  **Сортировка отказов (Failure Triage, FT):** Классификация типа сбоя для назначения нужной команде инженеров.
3.  **Локализация первопричины (Root Cause Localization, RCL):** Поиск конкретного сервиса или экземпляра, вызвавшего сбой.

**Проблема существующих подходов:**
Большинство существующих методов решают эти задачи изолированно. Это приводит к дублированию усилий (отдельная обработка данных, обучение моделей), снижению эффективности, задержкам в реагировании и отсутствию системного взгляда. Попытки создания мультитаковых методов часто требуют разметки данных (supervised learning), что дорого и сложно на практике.

### 3. Предлагаемое решение: Фреймворк ART
Авторы предлагают **ART** — первую сквозную неконтролируемую (unsupervised) платформу, объединяющую все три задачи. Ключевая идея заключается в извлечении **общих знаний (shared knowledge)**, заложенных в "аномальных отклонениях" (anomalous deviations).

Фреймворк состоит из трех модулей:

#### Модуль 1: Изучение состояния с учетом зависимостей (Dependency-Aware Status Learning)
Для точного предсказания "нормального" поведения системы модель последовательно учитывает три типа зависимостей (порядок важен — от мелкого к крупному):
1.  **Канальная зависимость (Channel Dependency):** Связи между метриками, логами и трейсами. Используется **Transformer Encoder**.
2.  **Временнáя зависимость (Temporal Dependency):** Изменения во времени. Используется **GRU**.
3.  **Зависимость вызовов (Call Dependency):** Топология взаимодействия сервисов. Используется **GraphSAGE**.

Модель обучается на нормальных данных (самоконтролируемое обучение), чтобы предсказывать состояние системы на следующем шаге.

#### Модуль 2: Получение единого представления отказа (Unified Failure Representation Acquisition)
На основе разницы между предсказанным (нормальным) и реальным состоянием вычисляются **отклонения**.
*   **ILD (Instance-Level Deviation):** Вектор отклонений для каждого экземпляра (сервиса).
*   **SLD (System-Level Deviation):** Агрегированный вектор отклонений всей системы.

Эти представления интерпретируемы, так как каждый элемент вектора соответствует конкретному каналу данных (например, загрузке CPU или количеству ошибок в логах).

#### Модуль 3: Неконтролируемое решение задач диагностики
Используя ILD и SLD, задачи решаются без размеченных данных:
*   **AD (Обнаружение):** Используется теория экстремальных значений (EVT) для определения порога аномальности на основе нормы SLD.
*   **FT (Сортировка):** Используется кастомный алгоритм кластеризации на основе "дерева разрезов" (cut-tree), который группирует отказы по характеру отклонений в каналах.
*   **RCL (Локализация):** Вычисляется косинусное сходство между ILD экземпляра и SLD системы. Экземпляры, чьи паттерны отклонений наиболее похожи на системные, считаются первопричиной.

### 4. Ключевые находки и обоснование
Авторы провели эмпирическое исследование, чтобы доказать наличие общих знаний в отклонениях:
*   **Для AD:** Норма SLD во время сбоев значительно выше, чем в нормальное время (в среднем на 22%).
*   **Для FT:** Разные типы сбоев (сеть, CPU, диск) характеризуются высокими отклонениями в специфических каналах данных.
*   **Для RCL:** Первопричины (Root Causes) имеют более высокую корреляцию (сходство) с системными отклонениями (SLD), чем пострадавшие, но не виновные сервисы.

### 5. Экспериментальная оценка
Исследование проводилось на двух наборах данных:
1.  **D1:** Эмуляция e-commerce системы (46 экземпляров, 210 сбоев).
2.  **D2:** Реальная система управления коммерческого банка (18 экземпляров, 133 сбоя).

**Результаты:**
ART превзошел современные методы (бейзлайны) по всем трем задачам:
*   **AD:** Улучшение F1-score на **5.65% – 60.8%**.
*   **FT:** Улучшение на **13.2% – 95.7%**.
*   **RCL:** Улучшение точности локализации на **13.3% – 205%**.

Также был проведен абляционный анализ, подтвердивший важность учета всех трех типов зависимостей и правильного порядка их моделирования.

### 6. Вклад и значимость
1.  **Новизна:** Первая end-to-end неконтролируемая платформа, объединяющая AD, FT и RCL.
2.  **Интерпретируемость:** Внедрение семантической информации (привязка к каналам данных) в абстрактные векторы признаков.
3.  **Эффективность:** Единая модель для всех задач снижает сложность эксплуатации по сравнению с набором разрозненных моделей.

### Заключение
Статья представляет значимый шаг вперед в области AIOps. ART предлагает элегантное и эффективное решение проблемы фрагментации инструментов управления инцидентами. Подход, основанный на анализе отклонений и едином представлении состояния, доказывает свою высокую эффективность на практике, устраняя необходимость в трудоемкой разметке данных для обучения.
